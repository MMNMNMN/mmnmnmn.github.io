太长 不看！

发表于 2018-08-01 | 更新于 2018-08-02 | 分类于 笔 记 | 浏览量： 次 | 评论数：

本文主要絮絮叨叨地说下我建站的过程以及心得。高中毕业后就没有写过作文，表达上会有些问题，请见谅。


技术支持

hexo + Github Pages，小白、懒人和白嫖党的福音。
hexo，为搭建静态博客而生，博客开发神器。懒人式开发，傻瓜式部署，你甚至不需要任何前端知识，html、css、js都不会也没有问题，只需要会编辑Markdown就行，美化、使用教程百度资料一大堆，只要照着做，ctrl + c、ctrl + v就行，部署只需要一个命令，简单粗暴无脑。
Github Pages是面向用户、组织和项目开放的公共静态页面搭建托管服务，站点可以被免费托管在Github上，拥有300m免费空间，甚至不需要域名，可长期使用。


建站历程

生命在于运动 码农在于折腾
先从我的第一个博客开始说吧。这个月中旬开始搞，到现在已经半个月了，期间我总共做了三个博客。
第一个完全是我自己做的，白手起家。刚开始干劲满满，脑子里有很多骚想法骚操作，弄了几天发现完全不行。最大的问题就是太丑，拿不出手，太丢人了。我没有学过设计，界面UI设计毫无章法。对颜色也不敏感，所以配不出好看的颜色。这个是硬伤，暂时解决不了。第二个问题就是浏览器的兼容问题。每个人的浏览器不同，屏幕分辨率不同，页面的显示就成了问题，可能在我的电脑上能正常显示的页面换台电脑就出问题了。我不是专业的前端工程师，这个问题也解决不了。所以第一个博客宣告失败。

前一个博客的主要问题出在了页面上，这个问题最好的解决办法就是用模板。
在做第一个博客的时候，我当时干劲十足，什么都想自己做，坚决不用别人的东西，坚决不用模板，然后现在就在津津有味地找模板了，真香.gif(我王境泽，就是从零开始学，通宵做页面，也不会用别人的东西！模板真好用，下次我还要)

先贴几张图吧


















这个博客基于Python2.7和Django开发，前端Layui + Element + jQuery，界面比上一个好太多了。原本我是想把这个部署到服务器，但是因为很多原因：域名购买和长期租用云服务器都是一笔不小的开支，在国内注册网站涉及到繁琐的备案问题，部署到服务器涉及到技术问题，没有后台管理系统不利于网站的维护。最主要的还是网站基本只有我一个人访问，付出与收益不成比例，综合考虑还是放弃了。这个博客我花了很多时间，最终还是没能把他展示出来给大家看，真的蛮遗憾的

第三个就是当前这个，其实一开始用hexo我是拒绝的，因为用hexo搭建博客与我的初衷有些违背。我是想边学边做，学以致用，做一个完整的网站。而hexo仅仅是为了做博客，不需要太多的技术，没有数据库，没有后台，这与真实的网站不符。不过既然选择了它就要做好。

我不知道做个人网站的这股子热情我还能持续多久，其实现在我已经有些厌倦了。所有东西都要靠自己摸索，没有人支持我，不论是技术上，还是精神上。最初的创作动力来源于对新事物的新鲜感，失去了新鲜感之后就开始迷茫、开始厌烦，我到底还要不要继续做下去，不想做了好累啊。从最开始的干劲十足，到现在的心灰意冷，我真的没想到才过了半个月我的心态会发生如此大的变化，可能我也是一个三分钟热度的人吧。
